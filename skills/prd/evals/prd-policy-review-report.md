# PRD 정책 분석 리포트

---

## Executive Summary

**"수강권 선물하기 기능" PRD**를 코드베이스의 9개 정책 문서와 대조 분석한 결과, **4건의 CRITICAL, 6건의 HIGH, 5건의 MEDIUM 수준 정책 충돌/모호성**을 발견했습니다. 가장 심각한 문제는 (1) 기존 수강권 시스템에 소유권 이전 메커니즘이 전혀 없다는 점, (2) 무제한권 기간 분할이 현재 아키텍처와 근본적으로 비호환이라는 점, (3) PRD 내부에서 FR-2(독립 만료)와 FR-4(구독 해지 시 즉시 만료)가 논리적으로 모순된다는 점입니다. PM은 이 PRD를 개발팀에 전달하기 전에 최소 4건의 CRITICAL 항목에 대한 의사결정을 완료해야 합니다.

## 분석 개요

- **분석 대상**: 수강권 선물하기 기능 PRD
- **분석 일시**: 2026-03-14
- **분석 방법**: 3개 관점 다관점 분석 + Socratic 검증
- **참고 문서**: 9개 정책 문서 참조
- **분석 관점**: 수강권 생명주기/소유권, 쿠폰/구독 해지 연동, 어뷰징 방지/운영 제한

## 정책 충돌 상세

각 발견된 충돌/모호성을 상세히 설명합니다.

### [CRITICAL-1] 무제한권(UNLIMIT) 기간 분할의 근본적 비호환성

**PRD에서**: FR-1 — "무제한권의 경우 기간을 분할하여 선물할 수 있다 (예: 30일 중 7일을 선물)"

**기존 정책에서**: 무제한 수강권은 `nPurchased=999`로 "횟수 무제한"을 표현하며, 기간 자체를 분할하는 개념이 없습니다. 기간은 `startDate~expireDate`로 단일 구간으로 관리됩니다. (출처: 수강권 도메인 정책:eventType 정책, 수강권 비즈니스 정책:UNLIMIT 정의)

**충돌/모호성**: 무제한권의 '기간 분할'은 기존 수강권의 횟수 기반 관리 체계와 근본적으로 다른 패러다임입니다. 30일 중 7일을 분리하면 원본 수강권의 연속성이 깨지며, 원본은 어떻게 되는지(23일로 줄어드는지, 그대로 유지되는지)도 불명확합니다.

**PM이 결정할 사항**:
- [ ] 무제한권 선물을 기능 범위에서 제외할지
- [ ] '기간 분할' 대신 '일정 기간의 신규 무제한권 생성'으로 재정의할지
- [ ] 기간 분할 시 원본 수강권의 잔여 기간 처리 규칙

**신뢰도**: ✅ 검증됨 (0.861)

### [CRITICAL-2] FR-2(독립 만료)와 FR-4(구독 해지 시 즉시 만료)의 내부 모순

**PRD에서**: FR-2 — "선물받은 수강권은 원래 수강권의 만료일과 **무관하게 독립적으로** 운영된다" / FR-4 — "선물받은 사용자가 구독을 해지하면 선물 수강권도 함께 **즉시 만료**된다"

**기존 정책에서**: 해당 없음 (PRD 내부 모순)

**충돌/모호성**: FR-2에서 선물 수강권이 '독립적으로 운영'된다고 하면서, FR-4에서 받은 사람의 구독 해지에 따라 즉시 만료된다고 합니다. 독립적이라면 구독 상태와 무관해야 하고, 구독에 연동된다면 독립적이 아닙니다. 이 두 요구사항은 논리적으로 양립할 수 없습니다.

**PM이 결정할 사항**:
- [ ] 선물 수강권을 완전히 독립적으로 운영할지 (60일 만료만 적용, 구독 해지와 무관)
- [ ] 선물 수강권을 받은 사람의 구독에 연동할지 (구독 해지 시 만료, 60일 독립 만료 삭제)

**신뢰도**: ✅ 검증됨 (0.89)

### [CRITICAL-3] 무기한 쿠폰이 기존 쿠폰 시스템과 비호환

**PRD에서**: FR-3 — "쿠폰 유효기간은 제한 없음(무기한)"

**기존 정책에서**: 쿠폰은 반드시 사용 기간(utcUseStart/utcUseEnd)을 가지거나, 상대 기간(useDay)으로 설정됩니다. 쿠폰 표시 상태(DisplayStatus) 계산 로직이 `utcUseEnd`를 기준으로 EXPIRED/NORMAL/PENDING을 판단하므로, `utcUseEnd`가 없으면 시스템 오류가 발생할 수 있습니다. (출처: 쿠폰 정책 문서:DisplayStatus 계산 로직, 사용 기간 설정)

**충돌/모호성**: 무기한 쿠폰은 기존 쿠폰 시스템이 전제하는 '유한한 사용 기간' 모델과 충돌합니다. 또한 무기한 쿠폰은 회계 처리 관점에서도 부채로 영구 잡히는 문제가 있습니다.

**PM이 결정할 사항**:
- [ ] 무기한 대신 충분히 긴 유효기간(예: 365일)으로 변경할지
- [ ] 무기한을 유지한다면 시스템 변경 비용을 감안할지

**신뢰도**: ✅ 검증됨 (0.89)

---

### [CRITICAL-4] 수강권 소유권 이전 메커니즘 부재

**PRD에서**: FR-1 — "사용자는 자신이 보유한 모든 타입의 수강권을 다른 사용자에게 선물할 수 있다"

**기존 정책에서**: 수강권(Ticket)은 생성 시 `studentId`로 소유자가 고정되며, 소유권을 변경하는 API나 비즈니스 로직이 존재하지 않습니다. 수강권 타입(eventType)에도 GIFT 등 선물 관련 타입이 없습니다. (출처: 수강권 도메인 엔티티:Ticket 구조, 수강권 도메인 정책:eventType 정책)

**충돌/모호성**: 수강권 선물은 기존 시스템에 전혀 없는 새로운 개념이므로, 새로운 수강권 타입 정의와 양도 프로세스 설계가 필요합니다.

**PM이 결정할 사항**:
- [ ] 선물 수강권의 새 타입명(예: GIFT) 확정
- [ ] '원본 차감 후 신규 발급' 방식이 적절한지 검토

**신뢰도**: ✅ 검증됨 (0.861)

### [HIGH-1] 기존 구독 해지 정책과 선물 수강권 즉시 만료의 충돌

**PRD에서**: FR-4 — "선물받은 사용자가 구독을 해지하면 선물 수강권도 함께 즉시 만료된다"

**기존 정책에서**: 구독 해지 시 "해지 시점 이후 자동 결제 중지, 현재 구독 기간은 유지(환불 없음)". 즉, 해지해도 다음 결제일까지는 이용 가능합니다. (출처: 구독 플로우:구독 해지 플로우)

**충돌/모호성**: 기존 정책의 '잔여 기간 유지' 원칙과 PRD의 '즉시 만료'가 상충합니다. 사용자 관점에서 구독을 해지했을 뿐인데 선물받은 수강권이 예고 없이 사라지는 것은 부정적 경험입니다.

**PM이 결정할 사항**:
- [ ] 선물 수강권도 기존 정책과 동일하게 잔여 기간을 유지할지
- [ ] 즉시 만료를 유지한다면 사용자에게 사전 고지가 필요한지

**신뢰도**: ✅ 검증됨 (0.89)

---

### [HIGH-2] '모든 구독 상품' 적용 범위의 모호성

**PRD에서**: FR-3 — "이 쿠폰은 모든 구독 상품에 적용 가능하다"

**기존 정책에서**: 구독 상품의 결제 유형은 6가지(SUBSCRIBE, LUMP_SUM, TRIAL, EXTEND, BONUS, IPAD)입니다. 쿠폰의 적용 조건(applyCondition)으로 세밀하게 대상을 제어합니다. (출처: 구독 도메인 정책:결제 유형, 쿠폰 정책 문서:적용 조건)

**충돌/모호성**: '모든 구독 상품'에 무료 체험권(TRIAL)이나 무료 보상권(BONUS)도 포함되면, 0원 상품에 30% 할인이 적용되는 비정상적인 상황이 발생합니다.

**PM이 결정할 사항**:
- [ ] 쿠폰 적용 대상을 유료 구독(SUBSCRIBE, LUMP_SUM, EXTEND)으로 한정할지
- [ ] '모든 구독 상품'의 정확한 범위를 명시할지

**신뢰도**: ✅ 검증됨 (0.89)

### [HIGH-3] '첫 결제' 정의의 모호성

**PRD에서**: FR-3 — "수강권을 선물받는 사용자에게 첫 결제 시 30% 할인 쿠폰을 자동 발급한다"

**기존 정책에서**: 기존 유사 쿠폰은 트리거 이벤트가 명확합니다 — WELCOME(회원가입 시), AFTER_TRIAL(체험 수업 완료 후). (출처: 쿠폰 정책 문서:쿠폰 타입)

**충돌/모호성**: '첫 결제'가 (A) 선물 수신 후 최초 유료 구독, (B) 해당 서비스 생애 최초 유료 결제, (C) 체험 후 첫 정규 구독 중 어느 것인지에 따라 대상 사용자가 완전히 달라집니다. 이미 유료 구독 중인 사용자가 선물을 받으면 쿠폰이 발급되는지도 불확실합니다.

**PM이 결정할 사항**:
- [ ] '첫 결제'의 정확한 정의
- [ ] 이미 유료 구독 중인 사용자에 대한 쿠폰 발급 여부

**신뢰도**: ✅ 검증됨 (0.89)

### [HIGH-4] 수강권 세탁(만료 연장) 악용 가능성

**PRD에서**: FR-2 — 선물 시점부터 60일 만료 / FR-5 — 같은 사용자에게 월 3회 선물 가능

**기존 정책에서**: 수강권 만료 시 잔여 횟수 소멸(환불 없음). 보너스 수강권만 회수 가능. (출처: 수강권 비즈니스 정책:만료 규칙)

**충돌/모호성**: 만료 임박한 수강권을 부계정에 선물하면 60일 새 만료가 부여됩니다. 부계정에서 다시 원래 계정으로 선물하면 무한 연장이 가능합니다. 부계정 생성 비용이 매우 낮으므로(소셜 로그인 1개면 충분) 실질적인 위협이 됩니다.

**PM이 결정할 사항**:
- [ ] 선물 수강권의 재선물을 금지할지 (1회 양도 제한)
- [ ] 선물 수강권 만료일을 원본 만료일과 60일 중 더 빠른 날짜로 설정할지

**신뢰도**: ✅ 검증됨 (0.86)

### [HIGH-5] 사용자 간 거래 횟수 추적 메커니즘 부재

**PRD에서**: FR-5 — "사용자당 월 최대 10회 선물 가능, 같은 사용자에게는 월 최대 3회"

**기존 정책에서**: 사용자 간 거래 횟수를 추적하는 메커니즘이 없습니다. 기존 쿠폰의 수량 제한(pubLimit)은 쿠폰 템플릿 전체의 발급 수량을 제한하는 것이며, 사용자 간 관계 기반 제한이 아닙니다. (출처: 쿠폰 정책 문서:수량 제한, 중복 발급 방지)

**충돌/모호성**: FR-5를 구현하려면 사용자 간 선물 이력을 추적하는 완전히 새로운 인프라가 필요합니다.

**PM이 결정할 사항**:
- [ ] 선물 이력 추적 범위와 보관 기간 결정
- [ ] 월 기준 초기화 시점 결정 (매월 1일 vs 롤링 30일)

**신뢰도**: ✅ 검증됨 (0.86)

### [MEDIUM-1] 선물 수강권의 커리큘럼/언어 타입 결정 규칙 미정의

**PRD에서**: 미정의

**기존 정책에서**: 수강권은 구독의 커리큘럼 타입(BASIC, BUSINESS, SMART_TALK)과 언어 타입(EN, JP)을 상속받으며, 수업 예약 시 이 타입이 일치해야 합니다. (출처: 수강권 도메인 정책:필터링 정책)

**충돌/모호성**: 보낸 사람이 비즈니스 영어 수강권을 선물하면, 받는 사람은 비즈니스 영어 수업만 들을 수 있는지, 일반 영어도 가능한지 불명확합니다.

**PM이 결정할 사항**:
- [ ] 선물 수강권이 원본의 커리큘럼/언어를 유지할지
- [ ] 받는 사람이 커리큘럼/언어를 선택할 수 있게 할지

**신뢰도**: ✅ 검증됨 (0.861)

### [MEDIUM-2] 수강권 사용 우선순위에서 선물 수강권 위치 미정의

**PRD에서**: 미정의

**기존 정책에서**: 수강권 우선순위: UNLIMIT/COUNT > PODO_TRIAL > 기타. (출처: 수강권 도메인 정책:우선순위 정책)

**충돌/모호성**: 사용자가 정규 수강권과 선물 수강권을 동시에 보유할 때, 어느 것을 먼저 사용할지 결정되지 않았습니다.

**PM이 결정할 사항**:
- [ ] 선물 수강권의 우선순위 결정 (정규권 이후 사용이 일반적)

**신뢰도**: ✅ 검증됨 (0.861)

### [MEDIUM-3] IP 기반 차단 정책의 실효성 문제

**PRD에서**: NFR-2 — "동일 IP에서 24시간 내 5회 이상 선물 시 자동 차단"

**기존 정책에서**: 현재 시스템에 IP 기반 rate limiting이 구현되어 있지 않습니다. (출처: 알림 도메인 정책:발송 제한 정책)

**충돌/모호성**: VPN으로 우회 가능하고, 공유 WiFi에서는 정상 사용자가 차단될 수 있습니다.

**PM이 결정할 사항**:
- [ ] IP 차단 외 추가 보안 수단(디바이스 핑거프린팅 등) 검토 여부
- [ ] IP 제한 횟수를 더 관대하게 조정할지 (예: 20회)

**신뢰도**: ✅ 검증됨 (0.86)

### [MEDIUM-4] 선물 쿠폰 자동 발급 타입 부재

**충돌/모호성**: 새로운 쿠폰 타입(예: GIFT_RECEIVED)과 자동 발급 트리거 이벤트 정의가 필요합니다.

**PM이 결정할 사항**:

**신뢰도**: ✅ 검증됨 (0.89)

### [MEDIUM-5] 선물 수강권 횟수 차감과 무제한권의 비정합성

**PRD에서**: FR-1 — "선물 시 보유한 수강권에서 지정한 횟수만큼 차감된다" + "무제한권의 경우 기간을 분할하여 선물할 수 있다"

**기존 정책에서**: UNLIMIT 수강권은 nUsed 기록은 남지만 잔여 횟수 검증 없이 사용 가능합니다. (출처: 수강권 도메인 정책:차감 정책)

**충돌/모호성**: 회차권은 '횟수 차감', 무제한권은 '기간 분할'이라는 이중 선물 메커니즘은 정책 복잡도를 높이고 엣지케이스를 증가시킵니다.

**PM이 결정할 사항**:
- [ ] 선물 메커니즘을 단일화할지 (예: 모든 타입 횟수 기반)
- [ ] 타입별 선물 규칙을 별도로 정의할지

**신뢰도**: ✅ 검증됨 (0.861)

## PRD 내부 모호성

PRD 자체에서 발견된 모호하거나 자기모순적인 부분들입니다. 기존 정책과의 충돌이 아닌, PRD 내부의 문제입니다.

| # | 위치 | 모호한 내용 | 왜 문제인지 | 제안 |
|---|------|-----------|------------|------|
| 1 | FR-2 vs FR-4 | FR-2는 "독립적 운영", FR-4는 "구독 해지 시 즉시 만료" | 독립적이면서 구독에 의존하는 것은 논리적 모순 | 둘 중 하나를 수정. 독립 만료(60일) 또는 구독 연동 중 택일 |
| 2 | FR-1 | "모든 타입의 수강권" 선물 가능 | 체험권(PODO_TRIAL)도 선물 가능한지 불명확. 체험권은 1회용이며 선물의 취지와 맞지 않을 수 있음 | 선물 가능 수강권 타입을 명시적으로 나열 |
| 3 | FR-1 | 무제한권은 "기간 분할", 회차권은 "횟수 차감" | 이중 메커니즘으로 정책 복잡도 증가 | 단일 메커니즘으로 통일하거나 각각의 규칙을 상세 정의 |
| 4 | FR-3 | "첫 결제 시" | '첫 결제'의 기준 시점과 대상이 불명확 | "선물 수신 후 최초 유료 구독 결제" 등으로 구체화 |
| 5 | FR-3 | "모든 구독 상품에 적용 가능" | 무료 상품(체험, 보상)도 포함되는지 불명확 | "유료 구독 상품(정기구독, 일시불, 연장)"으로 한정 |
| 6 | FR-5 vs NFR-2 | 사용자 기반 제한(FR-5)과 IP 기반 제한(NFR-2) | 두 제한의 관계가 불명확. 동시 적용인지, IP는 보조 수단인지 | 제한 정책의 계층 구조를 명시 |

### 수강권 생명주기 및 소유권 정책
- **분석 범위**: 수강권 발급, 양도, 만료, 회수 정책
- **발견 건수**: 5건 (CRITICAL: 2, HIGH: 1, MEDIUM: 2)
- **핵심 발견**: 기존 수강권 시스템에 소유권 이전 메커니즘이 전혀 없으며, 무제한권 기간 분할은 현재 아키텍처와 근본적으로 비호환
- **검증 점수**: 0.861 (검증됨)

### 쿠폰 정책 및 구독 해지 연동
- **분석 범위**: 쿠폰 시스템 호환성, 구독 해지 시 선물 수강권 처리
- **발견 건수**: 6건 (CRITICAL: 2, HIGH: 3, MEDIUM: 1)
- **핵심 발견**: 무기한 쿠폰은 기존 시스템과 비호환이며, FR-2와 FR-4 사이에 논리적 모순이 존재
- **검증 점수**: 0.89 (검증됨)

### 어뷰징 방지 및 운영 제한 정책
- **분석 범위**: 선물 횟수 제한, IP 차단, 악용 시나리오
- **발견 건수**: 4건 (CRITICAL: 0, HIGH: 2, MEDIUM: 2)
- **핵심 발견**: 왕복 선물을 통한 수강권 세탁(만료 무한 연장)이 가능하며, 사용자 간 거래 추적 인프라가 전무
- **검증 점수**: 0.86 (검증됨)

## 관점 간 교차 분석

- **공통 발견**: 3개 관점 모두 '선물 수강권의 재선물 가능 여부'가 미정의임을 독립적으로 지적. 이는 세탁 방지(abuse-limits), 소유권 모델(ticket-lifecycle), 쿠폰 중복 발급(coupon-subscription) 세 영역에 동시에 영향을 미치는 핵심 미정의 정책.
- **관점 간 상충**: 없음. 3개 관점의 분석 결과가 일관되게 동일한 방향을 가리킴.
- **통합 인사이트**: FR-2의 '60일 독립 만료' 정책이 세 관점에서 모두 문제의 핵심으로 등장합니다. 소유권 관점에서는 기존 만료 체계와의 충돌, 구독 관점에서는 FR-4와의 모순, 어뷰징 관점에서는 세탁 악용의 근본 원인으로 작용합니다. **60일 독립 만료 정책의 재설계가 가장 파급력이 큰 단일 의사결정**입니다.

## 권고사항

### 즉시 조치 (PRD 수정 전 필수)
1. **FR-2와 FR-4의 모순 해소**: 선물 수강권의 만료 정책을 '60일 독립 만료(구독 해지 무관)' 또는 '구독 연동 만료' 중 하나로 통일
2. **무제한권 선물 방식 재정의**: '기간 분할' 대신 '일정 기간의 신규 무제한권 생성'으로 변경하거나, 무제한권을 선물 대상에서 제외
3. **무기한 쿠폰 대체**: 유효기간 365일 등 유한한 기간으로 변경 권장
4. **선물 수강권 재선물 정책 추가**: 세탁 방지를 위해 '선물 수강권은 재선물 불가(1회 양도 제한)' 규칙 추가

### 단기 조치 (PRD 확정 전 권장)
1. **'첫 결제' 정의 명확화**: "선물 수신 후 최초 유료 구독 결제"로 구체화
2. **쿠폰 적용 대상 명시**: "유료 구독 상품(SUBSCRIBE, LUMP_SUM, EXTEND)"으로 한정
3. **구독 해지 시 처리 방식 결정**: 기존 정책(잔여 기간 유지)과 일관되게 할지 검토
4. **선물 가능 수강권 타입 명시**: 체험권 포함 여부 결정

### 장기 고려 (정책 보완 차원)
1. **선물 이력 추적 인프라**: 사용자 간 선물 횟수를 관리하는 새로운 데이터 모델 설계
2. **어뷰징 방지 고도화**: IP 기반 차단의 한계를 보완하는 추가 수단(디바이스 핑거프린팅, 행동 패턴 분석) 검토
3. **선물 수강권 타입 정책**: GIFT eventType 정의, 우선순위, 커리큘럼/언어 규칙 등 포괄적 정책 수립
4. **선물 쿠폰 타입 정책**: GIFT_RECEIVED CouponType 정의, 중복 발급 규칙, 자동 발급 트리거 정의
